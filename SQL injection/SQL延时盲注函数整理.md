# SQL 核心函数与 Less-9 延时盲注解析

你希望我帮你系统整理 SQL 中 `if(条件,a,b)` 和 `sleep(5)` 这两个核心函数，尤其是结合 SQLi-Labs Less-9 延时盲注的场景说明它们的用法，我这就为你梳理得清晰易懂：

### 1. 条件判断函数：IF(条件, a, b)

#### 基本语法

```SQL

IF(condition, value_if_true, value_if_false)
```

#### 参数说明

|参数|说明|
|---|---|
|`condition`|要判断的 SQL 条件（比如 `length(database())=8`、`ascii(mid(xxx,1,1))=115`）|
|`value_if_true`|条件为 `TRUE`（真）时，函数返回的值/执行的操作|
|`value_if_false`|条件为 `FALSE`（假）时，函数返回的值/执行的操作|
#### 核心作用

在 SQL 中实现「二选一」的逻辑判断，是延时盲注的「判断核心」—— 只有当我们猜测的条件成立时，才触发后续的延时操作，否则不触发。

#### 实战场景（Less-9 示例）

```SQL

# 判断数据库名长度是否为8，是则执行sleep(5)，否则不执行
IF(length(database())=8, SLEEP(5), 1)

# 判断数据库名第1个字符的ASCII码是否为115（对应字符's'），是则延时5秒
IF(ASCII(MID(database(),1,1))=115, SLEEP(5), 1)
```

### 2. 延时函数：SLEEP(seconds)

#### 基本语法

```SQL

SLEEP(seconds)
```

#### 参数说明

|参数|说明|
|---|---|
|`seconds`|数值型参数，指定进程休眠的秒数（比如 `5` 表示休眠5秒，`3` 表示休眠3秒）|
#### 核心作用

让 SQL 执行进程暂停指定的秒数，是延时盲注的「反馈核心」—— 因为 Less-9 页面无任何显式回显，我们只能通过「页面加载是否延时」来判断前面的条件是否成立。

#### 实战场景（Less-9 示例）

```SQL

# 单独执行：让SQL进程休眠5秒，页面会延迟5秒加载
SLEEP(5)

# 结合IF使用（核心用法）：仅当条件成立时才休眠
IF(1=1, SLEEP(5), 1)  # 1=1为真，触发5秒延时
IF(1=2, SLEEP(5), 1)  # 1=2为假，不触发延时，直接返回1
```

### 

举个完整的 Less-9 注入 payload 拆解：

```SQL

?id=1' and IF(length(database())=8, SLEEP(5), 1)--+
```

- `1'`：闭合 SQL 语句中的单引号，制造注入点；

- `and`：连接我们的自定义条件；

- `IF(length(database())=8, SLEEP(5), 1)`：核心逻辑，判断数据库名长度是否为8，是则延时5秒；

- `--+`：注释掉后面的原生 SQL 代码，避免语法错误。

### 总结

1. `IF(条件,a,b)` 是**判断开关**：控制只有猜测的条件成立时，才执行延时操作；

2. `SLEEP(秒数)` 是**反馈信号**：通过页面加载延时，让我们感知到条件成立（无回显场景的关键）；

3. 两者结合是 Less-9 延时盲注的核心：用 IF 判断、用 SLEEP 反馈，逐步猜解数据库信息。
