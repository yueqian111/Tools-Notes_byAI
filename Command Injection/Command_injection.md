# 模块6：命令执行 RCE（第7周）💻

---

#### 原理

命令执行漏洞（RCE, Remote Code Execution）的核心原理是：**应用程序将用户可控的输入内容，直接拼接到系统命令字符串中并执行**。如果对输入没有做任何过滤或过滤不严格，攻击者就可以构造恶意输入，执行任意系统命令，从而控制服务器。

---

#### 必学内容

1. **命令拼接符**

这些符号用于在一条命令中插入或执行新的命令，是RCE攻击的核心工具：

- `|`：管道符，将前一个命令的输出作为后一个命令的输入，也可用于执行后续命令（如 `ping 127.0.0.1 | whoami`）

- `;`：分号，无论前一个命令是否成功，都会执行后面的命令（如 `ping 127.0.0.1; whoami`）

- `&`：后台执行符，前一个命令在后台运行，同时执行后面的命令（如 `ping 127.0.0.1 & whoami`）

- `&&`：逻辑与，只有前一个命令执行成功，才会执行后面的命令（如 `ping 127.0.0.1 && whoami`）

1. **基础命令**

这些是渗透测试中常用的信息收集命令：

- `whoami`：查看当前执行命令的用户身份

- `id`：查看当前用户的UID、GID、所属组等详细信息

- `ls`：列出当前目录下的文件和文件夹

- `cat`：读取并输出文件内容（常用于读取配置文件、源码等）

#### 实操：DVWA Command Execution Low

DVWA的Command Execution模块Low级别是无任何过滤的，是学习RCE的最佳入门场景。

**实操步骤：**

1. 进入DVWA，将安全级别设置为Low。

2. 进入Command Execution模块，在输入框中输入目标IP。

3. 构造恶意输入，利用命令拼接符执行任意命令：

    - 示例1：`127.0.0.1; whoami` → 执行`whoami`查看当前用户

    - 示例2：`127.0.0.1 && cat /etc/passwd` → 读取系统用户文件

    - 示例3：`127.0.0.1 | ls -la` → 列出当前目录文件
